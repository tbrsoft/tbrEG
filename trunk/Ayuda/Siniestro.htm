Registro de Siniestros

El procedimiento de registro de siniestros es cubierto en esta sección. Para registrar un siniestro el usuario puede acceder al formulario a traves de distintas opciones:

  • 1. Desde el menú: Siniestro -> Registrar Siniestro.
  • 2. Desde el teclado: Tecla F6.
  • 3. Desde el formulario Siniestros Pendientes: Botón “Registrar Nuevo Siniestro”.
  • 4. Llamada entrante: Si tiene instalado el plugin para registrar llamadas telefónicas, cuando ingrese una llamada se despliega un aviso desde el cual se puede seleccionar la opción Registrar Siniestro.

El Parte de Salida esta dividido en cinco pestañas:

  1. Principal
  2. Datos del siniestro.
  3. Personas Afectadas.
  4. Bienes Afectados.
  5. Colaboracion.

A continuacion se analizan cada unas de las pestañas.

1. Principal

En la pestaña principal se encuentran los datos correspondientes al cuartel, que incluyen entre otros:

  • Grado y Tipo del Siniestro.
  • Dotacion.
  • Tiempos.

 

Tiempos

El sistema permite registrar los tiempos en los que el cuartel recibe la llamada de emergencia, acude al lugar del hecho, regresa al cuartel, etc.

El proceso se puede resumir en los siguientes pasos:

  • El cuartel recibe una llamada.
  • El encargado envia un movil para una preinspeccion al lugar del siniestro. El encargado selecciona el boton Salida PreInspeccion para registrar la salida.
  • El movil enviado se comunica para notificar que arribo al lugar de la emergencia. En este paso el encargado selecciona Arribo PreInspeccion.

El paso siguiente depende de si fue un aviso real o una falsa alarma. En caso de que sea una falsa alarma se registra en observaciones. Si la llamada fue real se continua el registro del siniestro de la siguiente manera:

  • Se envia la dotacion, hecho que se registra mediante el botón Salida Dotacion.
  • Cuando la dotacion arriba al lugar se selecciona Arribo al Lugar (QTH).
  • Al momento de finalizar las tareas en el lugar se elige Registrar Liberación (VL).

Cabe aclarar que la Preinspeccion es opcional, se puede enviar directamente la dotacion y comenzar a registrar desde la Salida Dotacion.

El registro de los tiempos tambien puede ser realizado desde la Consulta de Siniestros.

2. Datos del Siniestro

En esta pestaña se encuentran los datos de ubicacion, personas que se comunicaron y referencias del siniestro.

 

Contemplando la situacion en la que varias personas se comunican para informar del mismo siniestro (como puede ser un accidente de transito o un incendio), el sistema cuenta con una lista de personas que llaman al cuartel, que suelen denominarse Solicitantes.

Para registrar un Solicitante se elige el boton + que se encuentra a la derecha de la lista. El sistema muestra la siguiente pantalla:

 

En la misma se ingrsan los datos del solicitante, permitiendo registrar la relacion del solicitante con el siniestro (testigo ocasional, propietario, conductor, etc..)

Nota
Los datos de los solicitantes, afectados vehiculos etc, no se guardan en la base de datos hasta que no se guarda el Siniestro. Es recomendable guardar el siniestro con los datos basicos y completar luego la carga de datos para evitar inconvenientes.


3. Personas Afectadas

En esta pestaña se encuentra una lista similar a la de Solicitantes, solo que aqui se registran los datos de las personas afectadas por el siniestro.

En caso de que el solicitante sea a la vez afectado por el siniestro (por ejemplo, en un accidente donde llama el dueño del vehiculo damnificado) se recomienda registrarlo como Solicitante aclarando en la relacion que es a la vez damnificado.

4. Bienes Afectados

En esta pestaña de registran todos los bienes que hayan resultado damnificados por el siniestro. El sistema permite registrar una lista de vehiculos, daños a la vivienda, daños a campos o baldio, junto con datos complementarios como conexion de gas y electricidad, e incluyendo datos del seguro si correspondiera.

 

5. Colaboracion

En esta ultima pestaña se registra la colaboracion recibida de otros Cuerpos de Bomberos, Policia y Centros Asistenciales.

 

En general, con el paso del tiempo el Cuartel colaborará y recibirá colaboración de Cuarteles de la zona. Para evitar la repetición de datos el sistema va almacenando los datos de colaboraciones recibidas.
Para registrar la colaboracion de un Cuerpo de Bomberos se selecciona el boton + a la derecha de la lista, y el sistema muestra el presente formulario.
A continuacion se debe seleccionar un Cuerpo de Bomberos. De no existir se elige la opcion Buscar que se encuentra a la derecha de la lista de Cuerpos, se registran los datos del Cuerpo del que se esta recibiendo la colaboración, se guardan los datos y luego se selecciona el Cuerpo registrado. Acto seguido se selecciona el Responsable del Cuerpo y la Unidad utilizada. Al igual que antes, si no se encuentran en la lista se elige la opción <Nuevo>, y se carga el nombre o la unidad segun corresponda. Por ultimo se pueden ingresar observaciones opcionalmente y luego el boton Aceptar envía los datos al formulario de Registro de Siniestro.



Hasta aqui se explico lo referente a la carga de Siniestros. Una vez guardado el siniestro el sistema permite realizar modificaciones sobre el mismo a traves de la opcion Modificar en Siniestros Pendientes.
